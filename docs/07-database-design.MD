# Database Design

## Purpose

This document defines the database structure for the AI Conversational Assessment Agent.

The database is designed around the business assessment rather than the conversation itself. The conversation is only one method of collecting information, while the assessment data represents the long-term business record.

---

# Design Principles

The database follows these principles:

- Assessment-centric design
- Normalized where appropriate
- JSONB for flexible multi-value fields
- AI generates insights, database stores facts
- Conversation history is preserved for auditing and analytics

# Database Design Decisions

## Why separate Assessment and AssessmentData?

Assessment represents the assessment session itself.

AssessmentData represents the business information collected during that session.

This separation allows future support for:

- Multiple assessments over time
- Assessment versioning
- Imported assessment data
- Historical comparisons

---

## Why store Conversation separately?

Conversation history enables:

- Conversation replay
- Audit logs
- Prompt improvement
- User behavior analysis
- Analytics

---

## Why use JSONB?

JSONB is used for flexible multi-value fields that may change over time.

Benefits include:

- Easier schema evolution
- Reduced number of relational tables
- Simpler queries
- Better support for AI-generated structured outputs
