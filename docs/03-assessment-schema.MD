# Assessment Schema

## Purpose

This document defines all information that the AI Conversational Assessment Agent must collect, derive, and generate during an assessment.

---

# Assessment Overview

```text
Assessment
│
├── Metadata
├── Property Profile
├── Business Stage
├── Operations
├── Technology
├── Pain Points
├── Business Goals
├── AI Readiness
└── Recommendations
```

---

# 1. Metadata

These fields are generated by the system.

| Field | Type | Required | Source |
|--------|------|----------|--------|
| assessment_id | UUID | ✅ | System |
| created_at | DateTime | ✅ | System |
| updated_at | DateTime | ✅ | System |
| status | Enum | ✅ | System |
| completion_percentage | Integer | ✅ | Calculated |

Status values:

- IN_PROGRESS
- COMPLETED
- ABANDONED

---

# 2. Property Profile

These questions are always asked at the beginning of the assessment.

| Field | Type | Required | Source |
|--------|------|----------|--------|
| property_name | String | ❌ | User |
| property_type | Enum | ✅ | User |
| location | String | ✅ | User |
| ownership_type | Enum | ❌ | User |
| total_units | Integer | ❌ | User |

Property Types

- Villa
- Hotel
- Apartment
- Boarding House (Kos)
- Guest House
- Resort
- Commercial Building
- Other

---

# 3. Business Stage

This is the first branching point in the conversation.

| Field | Type | Required | Source |
|--------|------|----------|--------|
| business_stage | Enum | ✅ | User |

Possible Values

- PLANNING
- IDLE_PROPERTY
- UNDERPERFORMING
- OPERATING_SUCCESSFULLY

Conversation Branches

## Planning

Fields

- target_launch_date
- investment_budget

## Idle Property

Fields

- reason_not_operating
- target_launch_date
- main_obstacle

## Underperforming

Fields

- occupancy_rate
- monthly_revenue
- average_room_rate
- booking_channels

## Operating Successfully

Fields

- expansion_plan
- automation_interest
